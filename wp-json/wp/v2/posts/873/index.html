If you are a web developer or a software engineer, it\u2019s most likely you are using Visual Studio code as your code editor. Yeah, it’s fantastic and easy to use. But I wondered when I found that there is no default shortcut key for most used options in git workflow, which pulls changes from upstream. So I decided to have a shortcut for pulling changes from upstream for both the current branches and different branches and also I though I should write a blog post around this just to let people know how to do that. So, let\u2019s get started\u2026.<\/p>\n\n\n\n

Steps<\/p>\n\n\n\n

    \n
  1. Open Visual Studio Code and go to File \u00e0 Preference \u00e0 Keyboard shortcuts alternatively you can press CTRL + K S to open it.
    <\/li>\n\n\n\n
  2. Once you open it, you will see a lot of options there. Basically, these are the various operations for which you can assign a keyboard shortcut. So instead of getting lost in all options, at the top, you will see a search bar where you can search for a particular action in our case we will type \u201cgit pull\u201d (without quotes)

    <\/li>\n\n\n\n
  3. Once you search that you will see the first 2 entries of \u201cgit pull\u201d and \u201cgit pull from\u201d. And next to it you will see an empty cell under the Key Bindings column. You can double-click on that and press the key that you want to assign for that particular operation. In my case I will assign git pull to \u201cALT + E\u201d and git pull from to \u201cCTRL + ALT + E\u201d as you can see below.




    So, this is all that we have to do. Now next time when you want to pull changes, instead of going the usual way, on the keyboard you can just press the shortcut key whatever you assign in the previous step.
    I know this isn\u2019t rocket science, but still, it saves some seconds and feels great and productive!
    In case you face any issues while performing the above steps, please let me know in the comment section below.
